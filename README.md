# 资金费率套利机器人

> 1x 杠杆 | 多币种 Tier 体系 | 30% 手续费返佣 | BNB 折扣

同时持有**现货多头**和**永续合约空头**，每 8 小时收取一次资金费率，价格中性、不方向押注。

---

## 收益测算（含返佣 + BNB 折扣）

### 核心假设

```
本金:           $10,000
实际套利资金:    $8,000（80%，多币种分散）
链上底仓:       $1,500（15%，Aave 等稳定收益）
应急储备:       $500（5%）
杠杆:           1x（不加杠杆）
返佣比例:       30%
手续费（BNB 折扣后）:
  现货 Taker    0.075%（原 0.10%）
  合约 Taker    0.036%（原 0.04%）
```

### 手续费明细（单仓 $4,000 为例）

```
单次开仓:
  现货 Taker:    $4,000 × 0.075%  =  $3.00
  合约 Taker:    $4,000 × 0.036%  =  $1.44
  小计:                               $4.44

单次平仓:                             $4.44

一轮开+平合计:                        $8.88
返佣 (30%):     $8.88 × 30%          = $2.66
净手续费:                              $6.22
```

### 场景一：保守（熊市 / 横盘）

```
平均费率:       0.01% / 8h
套利资金:       $8,000

每日费率收入:   $8,000 × 0.01% × 3 = $2.40
每月费率收入:   $2.40 × 30          = $72.00

换仓频率:       每月 2 次（2 个仓位各换 2 次）
每月净手续费:   $6.22 × 4           = $24.88

链上底仓收益:   $1,500 × 8% ÷ 12   = $10.00

每月净收入:     $72.00 - $24.88 + $10.00 = $57.12
┌──────────────────────────────────┐
│   年化收益:  $57.12 × 12 = $685  │
│   年化利率:  ≈ 6.9%              │
└──────────────────────────────────┘
```

## 项目结构

```
funding-arb-bot/
├── main.py               # 主控程序 + APScheduler 调度器
├── executor.py           # 下单引擎（Binance 官方 SDK，保护返佣）
├── base_executor.py      # 执行器抽象基类（定义接口 + 分批下单逻辑）
├── sim_executor.py       # 模拟执行器（SimulatedExecutor，不下真单）
├── monitor.py            # 费率 + 基差监控（ccxt 只读，不影响返佣）
├── screener.py           # 动态筛选器（多 Tier、评分、流动性过滤）
├── position.py           # 仓位管理 + SQLite 存储
├── capital.py            # 资金分配计算（CapitalPlan）
├── notifier.py           # Telegram 通知
├── results.py            # 结果报表 + CSV 导出
├── preflight.py          # 启动前环境 + 权限预检
├── config.example.yaml   # 配置模板
├── requirements.txt      # Python 依赖
├── deploy.sh             # 一键部署脚本（Linux + systemd）
├── docs/
│   └── plan.md           # 优化路线图（v5~v8）
├── TODO.md               # 开发计划追踪
├── arbitrage.db          # 实盘数据库（自动生成）
├── simulate.db           # 模拟数据库（自动生成）
└── logs/                 # 运行日志（自动生成）
```

## 运行模式

| 模式 | 命令 | 说明 |
|------|------|------|
| 监控 | `python main.py --monitor --capital 10000` | 只读扫描，无数据库，无下单 |
| 模拟 | `python main.py --simulate --capital 10000` | 虚拟下单，写入 `simulate.db` |
| 实盘 | `python main.py --capital 10000` | 真实下单，写入 `arbitrage.db` |

## 快速开始

```bash
# 1. 创建虚拟环境并安装依赖
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 2. 复制配置并填入 API Key
cp config.example.yaml config.yaml
# 编辑 config.yaml：填入 api_key、api_secret、Telegram token/chat_id

# 3. 创建 Telegram Bot（可选但推荐）
#    - 找 @BotFather，发送 /newbot，记录 bot_token
#    - 找 @userinfobot 获取你的 chat_id

# 4. 预检（只读权限即可，不需要交易权限）
python preflight.py --monitor

# 5a. 先跑模拟模式验证（建议至少跑 1 周）
python main.py --simulate --capital 10000

# 5b. 查看模拟结果
python results.py

# 6. 确认收益后切换实盘（需要 API Key 开启交易权限）
python preflight.py       # 完整预检
python main.py --capital 10000

# 服务器部署（Linux + systemd，后台运行 + 开机自启）
bash deploy.sh
```

## Coin Tier 体系

| Tier | 币种 | 特点 |
|------|------|------|
| T1 | BTC、ETH | 最高分配上限，最低费率门槛 |
| T2 | SOL、BNB、XRP 等 20 个主流币 | 中等分配，标准门槛 |
| T3 | 其余小市值币 | 最小分配，需 `--t3` 标志开启，有上市时长和市值约束 |

## 安全清单

```
[ ] API Key 只开启「现货交易 + 合约交易」权限
[ ] 关闭「提币权限」（防止 Key 泄露资金被转走）
[ ] 设置 IP 白名单（只允许服务器 IP）
[ ] config.yaml 已加入 .gitignore，不要提交到 Git
[ ] 服务器开启防火墙，只开放 SSH 端口
[ ] 定期检查 Telegram 通知是否正常
[ ] BNB 余额充足（低于 0.05 BNB 时 bot 会自动告警）
```
